> 이 문서는 Google Tag Manager(GTM)나 GTM 템플릿에 익숙하지 않은 분들을 위해 작성되었습니다.  
> 처음 시작하는 분도 이 가이드를 따라 단계별로 진행하시면 픽셀을 설치하고 이벤트를 추적할 수 있습니다.

## 1. GTM과 픽셀이란?

### Google Tag Manager (GTM)란?
GTM은 웹사이트 소스코드를 직접 수정하지 않고도 다양한 추적 코드(태그)를 관리할 수 있는 Google의 무료 도구입니다.  
한 번만 GTM 스니펫을 웹사이트에 삽입해 두면, 이후에는 GTM 웹 콘솔에서만 작업하면 됩니다.

### 픽셀(Pixel)이란?
픽셀은 사용자가 웹사이트에서 어떤 행동(클릭, 구매 등)을 했는지 추적하는 작은 추적 코드입니다.  
**Ptbwa Insight Pixel**은 Addirect 플랫폼으로 해당 데이터를 전송합니다.

### GTM 템플릿이란?
GTM 템플릿은 픽셀 설치 코드를 미리 패키징해 둔 것입니다.  
`template.tpl` 파일을 GTM에 한 번 가져오면, 이후에는 코드 없이 설정 화면에서만 픽셀을 관리할 수 있습니다.

## 2. 시작 전 준비사항

| 항목 | 확인 방법 |
|------|-----------|
| **GTM 계정** | [tagmanager.google.com](https://tagmanager.google.com) 에서 계정 및 컨테이너 생성 |
| **GTM 스니펫 설치** | 웹사이트 `<head>` 와 `<body>` 에 GTM 스니펫 코드가 삽입되어 있어야 함 |
| **Pixel ID** | Addirect 담당자로부터 `sk_` 로 시작하는 Pixel ID를 발급받아야 함 |
| **template.tpl 파일** | 이 저장소에서 `template.tpl` 파일을 다운로드 |

> **GTM 스니펫이란?**  
> GTM 사용 시 최초 1회만 개발자에게 요청해서 웹사이트 HTML에 삽입하는 코드입니다.  
> GTM 콘솔 → 관리 → Google Tag Manager 설치 에서 확인할 수 있습니다.

## 3. 템플릿 가져오기 (최초 1회)

GTM에 **Ptbwa Insight Pixel** 템플릿을 등록하는 과정입니다. 한 컨테이너에 한 번만 하면 됩니다.

### 단계별 진행

**1단계** — [tagmanager.google.com](https://tagmanager.google.com) 에 접속하여 해당 컨테이너를 선택합니다.

**2단계** — 왼쪽 메뉴에서 **템플릿**을 클릭합니다.

**3단계** — "태그 템플릿" 섹션에서 **새로 만들기** 버튼을 클릭합니다.

**4단계** — 새 템플릿 편집 화면이 열리면, 우측 상단의 **⋮ (점 세 개)** 메뉴를 클릭합니다.

**5단계** — **가져오기**를 선택하고, 다운로드한 `template.tpl` 파일을 선택합니다.

**6단계** — 템플릿 내용이 로드되면 우측 상단의 **저장** 버튼을 클릭합니다.

✅ 이제 "Ptbwa Insight Pixel" 템플릿이 이 컨테이너에서 사용 가능합니다.

## 4. 이벤트별 태그 설정 방법

이벤트 종류마다 **별도의 태그**를 생성합니다. 아래 각 항목을 참조하세요.

> **태그란?** GTM에서 특정 조건(트리거)이 충족될 때 실행되는 코드 단위를 말합니다.

### 공통 태그 생성 절차

1. 왼쪽 메뉴에서 **태그** → **새로 만들기** 클릭
2. **태그 구성** 영역 클릭 → **Ptbwa Insight Pixel** 선택
3. 이벤트 유형에 맞게 아래 설정 진행
4. **트리거** 영역에서 실행 조건 설정 ([5장 참조](#5-트리거-설정-방법))
5. 태그 이름 입력 후 **저장**

---

### 4.1 페이지뷰 (page_view)

사용자가 페이지를 방문할 때마다 실행됩니다. 가장 기본적인 이벤트입니다.

| 필드 | 값 |
|------|----|
| **Pixel ID** | Addirect에서 발급받은 ID (예: `sk_abc123...`) |
| **이벤트 유형** | `페이지뷰` |

**트리거**: `All Pages` (모든 페이지) 트리거를 연결합니다.

---

### 4.2 클릭 (click)

특정 버튼이나 링크를 클릭할 때 실행됩니다.

| 필드 | 값 |
|------|----|
| **Pixel ID** | Addirect에서 발급받은 ID |
| **이벤트 유형** | `클릭` |
| **Click ID** | 클릭하려는 HTML 요소의 `id` 속성 값 (예: `btn-apply`) |

**Click ID 찾는 법**: 웹 브라우저에서 해당 버튼에 마우스 우클릭 → **검사(Inspect)** → HTML에서 `id="..."` 값을 확인합니다.

**트리거**: 해당 요소의 클릭 트리거를 설정합니다 ([5장 참조](#5-트리거-설정-방법)).

---

### 4.3 스크롤 (scroll)

사용자가 페이지를 일정 깊이만큼 스크롤했을 때 실행됩니다.

| 필드 | 값 |
|------|----|
| **Pixel ID** | Addirect에서 발급받은 ID |
| **이벤트 유형** | `페이지스크롤` |
| **스크롤 좌표 Value** | 추적할 스크롤 깊이 (숫자, 단위: px 또는 % — GTM 변수 활용 권장) |

**팁**: GTM의 기본 제공 변수인 `{{Scroll Depth Threshold}}` 를 **스크롤 좌표 Value** 에 입력하면,  
스크롤 깊이 트리거와 연동하여 실제 스크롤 위치를 자동으로 전달할 수 있습니다.

**트리거**: "스크롤 깊이" 트리거를 설정합니다 ([5장 참조](#5-트리거-설정-방법)).

---

### 4.4 전환 (conversion)

구매, 주문, 회원가입, 로그인 등 핵심 행동을 추적합니다.

| 필드 | 값 | 표시 조건 |
|------|----|-----------|
| **Pixel ID** | Addirect에서 발급받은 ID | 항상 |
| **이벤트 유형** | `전환` | 항상 |
| **전환유형** | `purchase` / `order` / `signup` / `signin` 중 선택 | 전환 선택 시 |
| **전환값** | 구매/주문 금액 (숫자, 예: `49000`) | purchase, order 선택 시 |
| **전환 Meta Data** | 추가 정보 (문자열, 예: `{"coupon":"SAVE10"}`) | 전환 선택 시 (선택사항) |

**전환유형 안내**:

| 값 | 의미 | 전환값 필요 여부 |
|----|------|----------------|
| `purchase` | 결제 완료 | ✅ 필요 |
| `order` | 주문 접수 | ✅ 필요 |
| `signup` | 회원가입 완료 | ❌ 불필요 |
| `signin` | 로그인 완료 | ❌ 불필요 |

**전환값을 동적으로 전달하는 방법 (개발자)**:  
전환값은 페이지에 따라 달라지므로, GTM **데이터 영역 변수(Data Layer Variable)**를 활용하는 것을 권장합니다.

```js
// 결제 완료 페이지에서 dataLayer에 값을 push
window.dataLayer = window.dataLayer || [];
window.dataLayer.push({
  event: 'purchase_complete',
  purchase_amount: 49000,
  coupon_code: 'SAVE10'
});
```

GTM에서 `purchase_amount` 를 데이터 영역 변수로 만들고 **전환값** 필드에 연결합니다.

## 5. 트리거 설정 방법

트리거는 "어떤 조건일 때 태그를 실행할 것인지"를 정의합니다.

### 5.1 모든 페이지뷰 트리거 (page_view용)

1. 태그 편집 화면 하단 **트리거** 영역 클릭
2. **+** 버튼 클릭 → **페이지뷰** 유형 선택
3. 조건: **모든 페이지뷰** 선택 → 저장

### 5.2 클릭 트리거 (click용)

1. 트리거 유형: **모든 요소** 또는 **링크만** 선택
2. 조건 설정: `클릭 ID` **같음** `btn-apply` (HTML의 `id` 값)

> **GTM 변수 활성화 필요**: GTM 콘솔 → 변수 → 기본 제공 변수 설정 → `클릭 ID`, `클릭 클래스` 등을 활성화해야 사용 가능합니다.

### 5.3 스크롤 깊이 트리거 (scroll용)

1. 트리거 유형: **스크롤 깊이** 선택
2. **세로 스크롤 깊이** 체크
3. 단위: **퍼센트** / 임계값: `25,50,75,100` (쉼표로 구분 입력)
4. 실행 조건: `각 페이지에서 한 번`

### 5.4 전환 트리거 (conversion용)

전환 이벤트는 특정 페이지(예: `/order/complete`) 진입 시 실행하는 것이 일반적입니다.

1. 트리거 유형: **페이지뷰** 선택
2. 조건: `페이지 URL` **포함** `/order/complete`

또는 **맞춤 이벤트** 트리거를 사용합니다 (dataLayer.push 연동 시).

## 6. 미리보기로 동작 확인하기

게시 전에 반드시 **미리보기 모드**로 실제 동작을 확인하세요.

1. GTM 콘솔 우측 상단의 **미리보기** 버튼 클릭
2. 테스트할 웹사이트 URL 입력 후 **Connect** 클릭
3. 웹사이트가 열리고 하단에 **GTM 디버그 패널**이 나타납니다
4. 페이지를 이동하거나 버튼을 클릭하면서 태그가 실행되는지 확인합니다

**확인 포인트**:
- 태그 목록에 **Ptbwa Insight Pixel** 태그가 나타나는지 확인
- 태그 상태가 **✅ Fired** 인지 확인
- 태그를 클릭하면 전송된 파라미터(`pid`, `ev`, `cid` 등)를 확인할 수 있습니다

## 7. 게시(Publish)하기

미리보기에서 정상 동작을 확인했다면 변경사항을 게시합니다.

1. GTM 콘솔 우측 상단 **제출** 버튼 클릭
2. **버전 이름** 입력 (예: `Insight Pixel 초기 설치`)
3. **게시** 클릭

> ⚠️ 게시 전까지는 실제 사용자에게 태그가 적용되지 않습니다.  
> 반드시 미리보기에서 확인 후 게시하세요.

## 8. 자동 수집 데이터 안내

픽셀은 태그 실행 시 아래 데이터를 **자동으로 수집**합니다. 별도 설정이 필요 없습니다.

| 데이터 항목 | 설명 | 보존 기간 |
|-------------|------|-----------|
| `cid` | 사용자 고유 식별자 (쿠키 기반, 자동 생성) | 30일 |
| `gid` | Google Analytics 사용자 ID (`_ga` 쿠키) | GA 설정 따름 |
| `gclid` | Google Ads 클릭 ID (URL 파라미터에서 자동 캡처) | 90일 |
| `url` | 현재 페이지 URL | — |
| `ref` | 유입 페이지(레퍼러) URL | — |
| `ts` | 이벤트 발생 타임스탬프 | — |
| `utm_*` | UTM 파라미터 (utm_source, utm_medium 등) | — |

### gclid 자동 처리
Google Ads에서 유입된 사용자의 URL에는 `gclid` 파라미터가 포함됩니다.  
픽셀이 이 값을 자동으로 감지하고 `_insight_gclid` 쿠키에 저장(90일)하여,  
이후 방문에서도 광고 클릭 정보가 유지됩니다.

## 9. FAQ / 자주 하는 실수

**Q. 태그가 "Fired"로 보이지 않습니다.**  
A. 트리거 조건을 확인하세요. 페이지뷰 태그라면 "All Pages" 트리거가 연결되어 있는지 확인합니다.

**Q. Pixel ID를 어디서 받나요?**  
A. Addirect 담당자에게 문의하시면 `sk_` 로 시작하는 Pixel ID를 발급받을 수 있습니다.

**Q. 같은 이벤트를 여러 페이지에서 다르게 추적하고 싶습니다.**  
A. 태그를 여러 개 만들고, 각 태그에 다른 트리거를 연결하면 됩니다.

**Q. 미리보기에서는 잘 작동하는데 게시 후 작동하지 않습니다.**  
A. 게시 후 브라우저 캐시를 비우고 다시 확인하세요. 또는 시크릿 창에서 테스트합니다.

**Q. 전환값은 어떤 형식으로 입력하나요?**  
A. 숫자만 입력합니다. (예: `49000`) 쉼표나 통화 단위(`원`, `₩`)는 포함하지 않습니다.

**Q. GTM 스니펫은 직접 설치해야 하나요?**  
A. 최초 1회는 웹사이트 소스코드에 스니펫을 삽입해야 합니다. 개발자 또는 웹사이트 관리자에게 요청하세요.  
이후 모든 태그 관리는 GTM 콘솔에서만 진행합니다.

**Q. 여러 이벤트를 동시에 설정할 수 있나요?**  
A. 네, 이벤트 유형마다 별도의 태그를 만들어 각각 트리거를 연결하면 됩니다.
